# TinyNote 小笺 — 功能总结（v1.0.0）

## 应用定位

TinyNote（小笺）是一款**轻量 Windows 桌面待办管理应用**，基于 Tauri 2 + React + TypeScript + SQLite 构建。主打简洁的毛玻璃界面、本地离线可用、贴桌常驻。适合随手记录每日小事、推进跨日长期任务。

## 核心功能

### 1. 日期视图
- 顶部日期条展示未来 N 天（默认 7 天，可在设置中选 3/7/14），左右箭头逐日翻页，"今天"按钮一键回到今日
- 有活跃任务的日期显示红点；今日日期高亮区分
- 切换日期时选中标签自动滚动到可视区域中央
- 选中日期的任务分两组展示：未完成 + 已完成

### 2. 任务管理
**三种任务类型**：

| 行为 | 普通 (manual) | 每日 (daily) | 多日 (multi_day) |
|------|:--:|:--:|:--:|
| 跨日显示 | 仅当天 | [开始, 结束] 范围 | [开始, 结束] 范围 |
| 进度顺延 | ❌ 永不 | ❌ 每天清零 | ✅ 当日及之前继承 |
| 完成机制 | 改任务自身状态 | 写 per-date progress entry | 写 per-date progress entry |

**任务操作**：
- 底部栏快速添加普通任务（输入标题 + Enter）
- 点击任务标题内联编辑
- 拖动进度条调整进度（0–100%）
- 勾选完成 / 恢复
- 右键或更多菜单：延期、任务详情、归档、删除（均带二次确认）
- 批量"清空已完成"（带确认弹窗）

### 3. 顺延（Postpone）机制
- 单任务延期：在任务菜单选择目标日期，将任务顺延到该日（manual / multi_day 支持）
- 批量顺延：底部栏"顺延"按钮，推迟选定日期的活跃任务到次日
  - manual 任务全部符合
  - multi_day 任务仅当处于结束当天时符合（顺延即延长结束日期）
  - daily 任务每天清零，不支持顺延

### 4. 任务管理面板
- 顶部入口（ListTodo 图标）打开，管理每日 / 多日任务
- 分段切换"每日"/"多日"视图
- 顶部固定新建区：标题 + 开始/结束日期，创建对应类型任务
- 列表行内编辑标题与日期、删除（带确认）
- 普通任务不在本面板管理（由底部栏快速添加）

### 5. 任务详情对话框
- 展示任务类型、状态、当前日期、开始/截止、创建/更新时间、最近延期
- 延期历史记录
- 高级排期：可修改任务类型、开始/截止日期

### 6. 归档面板
- 查看已完成/归档任务
- 恢复或永久删除（带确认）

### 7. 设置面板
- 展示天数（3/7/14）、完成归档开关、开机启动、启动显示、固定桌面、置顶
- 窗口透明度滑块
- 7 套主题（跟随系统、玻璃蓝/白/薄荷/紫、亮色、暗色）
- 字号、背景图片
- 恢复窗口、导出/导入 JSON

### 8. 窗口特性
- 无边框毛玻璃窗口，自定义标题栏可拖动
- 置顶、固定桌面（锁定位置）
- 关闭隐藏到系统托盘（非退出）
- 窗口状态记忆
- 系统托盘菜单：打开/隐藏、添加今日任务、固定桌面、置顶、开机启动、退出

### 9. 数据管理
- 本地 SQLite 存储，完全离线
- JSON 导出/导入（含任务、例行、进度、延期记录）
- 同步字段预留（user_id、device_id、sync_status、version）为后续多端同步做准备

## 技术栈

| 层级 | 技术 |
|------|------|
| 桌面壳 | Tauri 2 |
| 前端 | React 18 + TypeScript |
| 构建 | Vite 5 |
| 状态 | Zustand 5 |
| 数据库 | SQLite（@tauri-apps/plugin-sql） |
| 测试 | Vitest 2 + @testing-library/react |
| 图标 | lucide-react |

## 架构

```
Repository（SQL）→ Service（业务逻辑）→ Store（Zustand）→ Component（React UI）
```

- **Repositories**：通过 `@tauri-apps/plugin-sql` 执行 SQL，`db.ts` 为单例
- **Services**：业务逻辑层，部分纯函数（`taskWorkflow.ts`、`routineLogic.ts`、`taskOccurrence.ts`、`taskScheduling.ts`）易测试
- **Stores**：`taskStore`（任务/日期/CRUD/导航/顺延）、`settingsStore`（设置+原生开关）、`uiStore`（面板可见性）
- **任务可见性与进度继承**：核心在 `taskOccurrence.ts`，`shouldShowTaskOnDate` 控制可见性，`resolveInheritedProgressEntry` 控制多日进度顺延（仅当日及之前）

## 数据模型

5 张 SQLite 表：`tasks`、`routines`、`routine_instances`、`app_settings`、`sync_log`。所有写操作递增 `version`、设 `sync_status='pending'`、写 `sync_log`。

## 已知限制与后续规划

1. **daily 任务不可顺延**：每日进度清零的设计取舍，不支持顺延。
2. **批量顺延仅多日结束日**：multi_day 批量顺延仅在结束当天触发（延长结束日期）。
3. **routine 系统休眠**：例行任务（routines）数据可导出/导入，但 v1.0 无 UI 创建入口、自动生成管线未接入。后续评估补全或移除。
4. **无多端同步**：同步字段已预留，实际同步未实现。
5. **面板焦点管理**：键盘/读屏用户体验待增强（打开面板未自动聚焦）。
