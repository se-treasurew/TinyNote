### 关键修复说明

**BUG-1**：`postponeTask` 第 160 行原为 `percent: sourcePercent`，改为 `percent: existingNextEntry?.percent ?? sourcePercent`。新增测试"preserves existing target-date progress instead of overwriting it on postpone"覆盖此路径（原测试故意 mock 第二次 `findProgressEntry` 返回 null，未覆盖冲突场景）。

**BUG-2**：核实发现 `carryProgressForward` 设置已从 `AppSettings` 类型/默认值/设置面板中**完全移除**，但 CLAUDE.md 仍在描述它。经用户确认采用**硬编码顺延**方案：多日任务进度在"当日及之前"始终继承，未来日期不显示继承进度。同步修正 `taskOccurrence.ts`、`taskService.loadVisibleTasks`、`taskStore`（移除对已不存在的设置的读取）及相关测试。

**BUG-6**：在 `routineRepository` 新增 `deleteInstanceByTaskId(taskId)` 方法，并在 `taskService.deleteTask` 中当 `task.routineId` 存在时调用，释放 `routine_instances` 槽位使任务可重新生成。

## 五、版本号管理

### 问题根因
用户反馈"每次本地开发都递增不合理"。核实发现：**不存在任何自动递增机制**（无 CI、无 git hook、无 version-bump 脚本、`build.rs` 不碰版本）。漂移源于 `TitleBar.tsx:21` 的**手动硬编码版本字符串**，每次 UI 提交随手改（git 历史显示 0.1.0→0.1.2→0.1.5→0.1.6→0.1.11，跳号且配置文件长期滞后到 0.1.0，最近才统一到 0.1.11）。

### 修复方案
1. **版本号设为 `1.0.0`**（语义化版本，首个正式版）：`package.json`、`tauri.conf.json`、`Cargo.toml` 三处统一。
2. **TitleBar 改用 Tauri 运行时 `getVersion()`**（`@tauri-apps/api/app`），从 `tauri.conf.json` 读取，消除硬编码——单一真实来源，永不漂移。带 try/catch 守卫，非 Tauri 环境（测试/浏览器）不报错。
3. **CLAUDE.md 版本段更新**：移除"需同步更新 TitleBar 版本号"的旧指引，改为"版本以 `tauri.conf.json` 为准，TitleBar 运行时读取"。
4. **后续发版流程**：只改 `tauri.conf.json`（+ package.json/Cargo.toml 保持一致），TitleBar 自动跟随。

## 八、建议后续跟进（非第一版阻塞）

1. **routine 系统去留**：补全 routine 创建 UI 或彻底移除 routine 系统（含表、迁移、dataPortability 引用）。
2. **面板焦点管理**：打开面板/对话框时未 trap/move 焦点，键盘与读屏用户需手动 Tab。可后续加焦点管理。
3. **disabled 按钮提示**：顺延/清空按钮 disabled 时无 `title` 说明原因。
4. **加载态**：各面板（归档、任务管理、设置）数据加载时无 loading 指示。
5. **daily 顺延限制**：daily 任务不可顺延（设计取舍），可在 UI 加 tooltip 说明。
