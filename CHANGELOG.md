# TinyNote 更新日志

本项目的版本说明按发布版本维护。应用检查到新版本时，会显示对应 GitHub Release 中的更新说明。

## [1.2.4] - 2026-07-17

### 修复

- 修复已完成的多日任务仍持续出现在后续日期“已完成”区域的问题：现在以实际勾选完成的 occurrence 日期为边界，完成日及此前保留历史，完成日之后不再展示。
- 恢复已完成的多日任务时会清除完成边界，使任务重新显示到原截止日期；每日任务、普通任务和延期副本的既有完成规则保持不变。
- 已完成子任务在完成边界后的日期会隐藏，但父任务的子任务数量、完成进度和完成约束仍按完整任务树计算。

### 数据兼容

- 新增数据库 migration 10，为历史已完成多日任务回填并校正完成日期边界；旧版 JSON 备份导入时也会按相同规则补齐。

## [1.2.3] - 2026-07-17

### 修复

- 修复任务管理面板编辑每日或多日任务时，编辑表单高度未撑开列表行、遮挡后续任务的问题。
- 编辑任务现在会保持在正常的纵向列表流中，并在空间不足时自动滚动到可视区域；保存、取消和任务排期逻辑保持不变。

## [1.2.2] - 2026-07-10

### 修复

- 修复多日任务在某天完成后，后续日期又重新显示为待办的问题：现在完成任意一天的多日任务会完成整个任务；每日任务仍保持按天独立完成。
- 修复已完成普通任务被遗留的进度记录覆盖、再次显示为待办的问题。
- 修复普通任务延期到日期窗口内时，原始日期不在窗口便无法显示延期副本的问题。
- 修复一键顺延在父子任务级联时重复写入延期历史的问题；重复执行同一项顺延现在保持幂等。
- 修复父任务对子任务完成状态的判断：未完成直接子任务仍会阻止完成父任务；已全局完成的多日子任务或原始日期已完成的普通子任务，不会再被遗留进度误判为未完成，避免任务被错误归类。

### 数据兼容

- 升级时自动修复符合条件的历史多日完成状态，并合并重复的有效延期历史记录；不会回滚任务日期或独立进度。

## [1.2.1] - 2026-06-30

### 调整

- 任务进度条改为拖动结束后一次性保存，减少拖动过程中的重复写入；进度达到 100% 时自动完成当前日期的任务。
- 多日任务在列表中的类型标签改为显示截止日期，例如“截止 6月30日”；旧数据缺少截止日期时显示“截止未设置”。
- 更换应用图标，统一窗口、系统托盘、可执行文件和 Windows 安装包的品牌图形。
- 本次图标更新沿用 1.2.1 版本号；已经安装 v1.2.1 的用户不会收到自动更新，需要重新下载安装包才能获得新图标。

## [1.2.0] - 2026-06-27

### 新增

- 二级子任务：任务树支持最多三层（母任务 → 子任务 → 孙任务），孙任务可单独完成。母任务和子任务行可展开/折叠自己的子树（每节点独立箭头，非持久化）。
- 子任务完成自动推进主任务进度：完成或恢复一个子任务时，按直接子任务的完成比例自动重算并写入母任务（及更上层祖先）的进度。每日 / 多日母任务按当日进度比例推进，全部子任务完成时母任务自动标记完成；普通母任务在全部直接子任务完成时自动完成。
- 延期向上级联：延期任意一个子任务时，其母任务及更上层祖先一并延期到同一目标日期（原有「延期母任务连带子任务」的向下级联保留），整条分支同步移动。
- 透明度全范围调节：窗口透明度滑块扩展为 0%–100%，并显示当前百分比。可完全透明或完全不透明。

### 调整

- 进度条与徽标布局重构：只有叶子任务（无子任务的最末级）显示可拖动进度条；含子任务的任务改为在标题行右侧显示醒目的「已完成数/直接子任务数」徽标，不再显示进度条，且与「普通/每日/多日/延期」标签分行。
- 进度条移到任务行右侧独立列，更醒目，不再与类型标签挤在同一行。
- 延期、删除、完成等操作的级联改为递归覆盖全部后代（孙任务一并处理），并新增向上级联祖先。

### 修复

- 修复延期副本完成日期错误：例如 26 日任务延期到 27 日后，在 27 日勾选完成只会完成 27 日 occurrence，不会把 26 日原任务误标完成。
- 修复子任务添加态缺少可见取消入口的问题：新增确认/取消图标按钮，并在切换日期、折叠父任务或打开底部添加时自动退出子任务添加。
- 调整完成按钮内的勾选图标居中方式，避免 √ 在圆形按钮中偏斜。

## [1.1.0] - 2026-06-21

⚠️ **本版本含数据库底层变动，不支持从 1.1.0 降级回 1.0.0 / 1.0.1 / 1.0.2。** 升级到 1.1.0 后请继续使用 1.1.0 或更高版本，不要安装旧版安装包。详见下方「升级与降级」。

### 升级与降级

- **支持升级**：从 1.0.0 / 1.0.1 / 1.0.2 升级到 1.1.0 正常，应用内自动更新或手动安装 1.1.0 安装包均可，历史任务与设置保留。
- **不支持降级**：从 1.1.0 降级到任何旧版本会导致应用无法启动。原因是本版本新增了数据库迁移（`parent_task_id` 索引），旧版本在启动时会校验已应用的迁移必须全部存在于自身迁移列表中，遇到未知迁移即报错退出；同时 Tauri 安装器默认也拒绝覆盖更高版本。如需运行旧版本，请先卸载 1.1.0 并删除本地数据库后全新安装旧版（会丢失全部本地数据）。

### 新增

- 子任务功能：每个任务可在右键菜单中添加子任务，以缩进形式展示在母任务下方，并可单独完成；母任务行显示「x/y」完成进度徽标。
- 子任务自动继承母任务的类型与日期范围，每日 / 多日母任务的子任务随母任务在范围内逐日显示并按日独立完成。
- 删除母任务时级联软删除其全部子任务；单任务顺延时同步顺延同日的子任务。
- 母任务排期变更时，子任务跟随更新，保持任务树一致。
- 为 `parent_task_id` 新增数据库索引，提升子任务级联查询性能。

### 调整

- 移除「过期」标签：不再按发生日期是否已过显示过期标识。
- 已完成区计数改为按任务树统计，与缩进展示保持一致。

## [1.0.1] - 2026-06-19

### 新增

- 任务右键菜单支持取消延期，清除该任务的延期标识与全部延期历史。
- 关于面板在发现新版本时展示 GitHub Release 更新说明。

### 调整

- 移除归档面板、归档操作和“完成归档”设置，任务完成后统一进入“已完成”。
- 旧归档任务和进度记录自动迁移为已完成，并继续兼容旧版导入数据。
- 点击标题栏“小笺 + 版本号”打开关于面板，减少独立图标按钮。
- 移除底部“清空已完成”，已完成任务继续支持逐条恢复或删除。
- 更新项目说明、开发文档与 GitHub Release 发布流程。

## [1.0.0] - 2026-06-19

### 新增

- 普通、每日和多日三种任务类型，以及按日期浏览和快速添加。
- 每个 occurrence 独立保存进度，多日任务支持当日及历史进度继承。
- 单任务延期到指定日期和底部一键顺延，并记录完整延期历史。
- 任务详情与高级排期编辑。
- 毛玻璃主题、窗口置顶、固定桌面、最小化、托盘和开机启动。
- SQLite 本地存储、JSON 导入导出和同步字段预留。
- 关于面板、GitHub 链接、检查更新和签名安装更新。
